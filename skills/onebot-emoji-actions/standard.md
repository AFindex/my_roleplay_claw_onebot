# 标准 OneBot v11 表情相关能力

## 推荐优先支持

- `text`：直接发送普通 Unicode emoji，例如 `😊🥺😭`
- `face`：QQ 系统表情，格式：

```json
{
  "type": "face",
  "data": {
    "id": "123"
  }
}
```

- `image`：静态表情包、GIF、闪照都走图片段
- `rps`：猜拳魔法表情
- `dice`：骰子魔法表情
- `shake`：窗口抖动
- `poke`：戳一戳，格式：

```json
{
  "type": "poke",
  "data": {
    "type": "126",
    "id": "2003"
  }
}
```

## 在这个仓库里的建议

- 低层发送已经能发任意消息段数组，优先复用，不要重复造发送器
- 高层如果还只暴露文本 / 图片，先补一个“中间标记 -> 消息段”的转换层
- 入站解析时，不要只保留 `text`；至少把 `face` / `poke` 变成可读提示再喂给模型

## 保守策略

- 如果没有稳定的 `face` 映射表，优先使用普通 emoji，不强行指定系统表情编号
- 如果对端不支持复杂段，保留文字兜底，例如“她轻轻戳了你一下”
