# QQ / OneBot 表情风格

这个 skill 关注“怎么用表情才像这个角色”，不负责直接调用 OneBot / NapCat 接口。

## 适用场景

- 用户要给角色补一点表情感、互动感、QQ 味
- 用户要约束 emoji / QQ 表情的密度、强度和禁区
- 用户要设计“模型输出什么中间标记，后处理再转消息段”

## 核心原则

- 先保住人设和可读性，再补表情
- 没有明确平台段支持时，优先使用普通 Unicode emoji 或纯文字语气
- 不确定 `face.id` 时，不要硬猜 QQ 系统表情编号
- 表情要服务语气，不要把回复写成表情堆叠

## 密度建议

- 短回复：`0-2` 个 emoji / 表情动作
- 中等回复：最多 `3` 个
- 连续多轮对话里，至少保留一部分“纯文字”回复，避免每句都很花

## 选择顺序

1. 先判断强度：`none` / `light` / `medium` / `strong`
2. 再选载体：
   - 轻微情绪：标点、停顿、语气词、Unicode emoji
   - 明显 QQ 氛围：`face`、`poke`
   - 视觉主导：`image` / GIF 表情包
3. 最后补一个可读的文字兜底，不要只剩平台码

## 推荐输出格式

如果链路只支持文本，直接输出普通 emoji 即可。

如果后处理会把标记翻译成 OneBot 消息段，优先输出中间标记：

- `[[emoji:🥺]]`
- `[[face:123]]`
- `[[rps]]`
- `[[dice]]`
- `[[poke:type=126,id=2003]]`

除非当前任务就是兼容 CQ 码，否则不要直接手写 `[CQ:...]`。

## 风格模板

- 少女 / 轻柔：少量 `🥺` `✨` `///` `><`，更依赖停顿、语气词和轻微害羞感
- 冷静 / 克制：少量 `…` `🙂`，表情出现频率更低
- 俏皮 / 逗弄：`😉` `😼`、偶尔 `poke`，但不要连续刷互动动作
- 认真解释：优先纯文字，只在开头或结尾点一小下表情

## 协作方式

- 需要真的在这个插件里发出 `face` / `poke` / `mface` / 贴表情时，配合 `onebot-emoji-actions`
- 需要把角色人设和表情偏好写进模板时，把这里的“密度、偏好、禁区、占位标记”写清楚
